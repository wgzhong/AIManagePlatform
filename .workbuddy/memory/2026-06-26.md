## 贡献技能权限修复 v11-owner-fix (17:16)
### 问题
lupa 贡献的 test/test2 技能显示为"只读模式"，但贡献者本人应该可以编辑和取消贡献

### 核心改动
1. **新增 `getCurrentUid()` + `isMySkill(s)` 函数** — 从 localStorage 读取 user_id，与 skill.owner_id 比较
2. **pick() 只读判断逻辑变更**：`readonly && !isMySkill(s)` → 贡献者可编辑，其他人只读
3. **新增 contributed+isMine 加载路径**：从 `/api/user-skills/{name}/raw-md` 加载（可编辑）
4. **contribRow 条件扩展**：`source==='private'` 或 `(source==='contributed' && isMine)` 时显示贡献开关
5. **保存/删除 API 路由**：所有 save/delete 操作通过 `useUserSkillApi()` 判断走 user-skills 还是 skills
6. **删除按钮**：自己的贡献技能也显示删除按钮（之前被 readonly 隐藏）

### 权限矩阵
| 技能类型 | 所有者 | 他人 |
|---------|--------|------|
| global (系统内置) | admin可改 | 只读 |
| private (私人) | 可编辑+贡献开关 | 不可见 |
| contributed (自己) | **可编辑+取消贡献开关** | **只读** |
| contributed (他人) | 不适用 | 只读 |
