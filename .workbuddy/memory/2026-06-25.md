## API Key 脱敏截断 Bug 修复
- **现象**：用户在首页输入完整 API Key，但 `_attempt_stream` 收到的是脱敏值 `20ea25...jZeR`
- **根因**：GET `/auth/config` 出于安全返回 `mask_api_key()` 脱敏值，`chat.html` 的 `loadUserConfig()` 直接用脱敏值作为 `currentApiKey` 发送给 `/chat`，且把脱敏值写回 `localStorage` 覆盖了完整 key
- **前端修复**（`app/static/chat.html`）：
  - 新增 `isMaskedKey()` 检测脱敏值（`...` 或 `***`）
  - 新增 `getFullApiKeyFromStorage()` 从 `localStorage('ai_platform_api_key')` 和 `ai_chat_config.key` 恢复完整 key
  - `loadUserConfig()` 优先用 localStorage 完整 key，脱敏值仅用于判断是否已配置
  - 写回 localStorage 前检查非脱敏，避免污染
  - catch 块同样使用 `getFullApiKeyFromStorage()`
- **后端修复**（`app/api/user_config.py`）：PUT 响应也改为 `mask_api_key()`，与 GET 一致，避免响应体泄露完整 key

## 时间技能工具调用链路修复
- **现象**：用户问「几点了」，AI 回复"让我帮你看看现在几点啦！"但未调用 get_time 工具
- **根因**：多因素叠加
  1. 系统提示词中工具调用规则被放在末尾，被情绪人设协议压制
  2. `llm_stream.py:169` 有 `print("api_key=", api_key)` 安全漏洞
  3. chat_service.py 追加 role:tool 消息缺少 tool_call_id（不符合 OpenAI 规范）
  4. 前端工具结果正则把 emoji 和标点全删了
  5. get_time 技能 description 未足够强调「必须调用」
- **修复**：
  - 删除 api_key 明文 print（安全漏洞）
  - 重构 `_build_messages()`：工具调用规则提升为最高优先级，放在 system prompt 最前面，使用强制措辞
  - get_time 改为【强制调用】开头，明确告知 LLM 无内置时钟
  - 补全 OpenAI 规范：追加 assistant tool_calls 消息 + role:tool 带 tool_call_id
  - 前端移除过度正则过滤，保留 emoji 和中文标点
- **修改文件**：`llm_stream.py`, `chat_service.py`, `get_time/__init__.py`, `chat.html`
- **最终修复方案（2026-06-25 第三次修改）**：
  - **根因确认**：GLM API 对 `tool_choice` 参数的支持有问题，LLM 始终不返回 tool_calls
  - **最终方案**：关键词命中时**后端强制直接执行技能**，完全不依赖 LLM 返回 tool_calls
  - `chat_service.py` 新增 `_parse_skill_args()` 方法解析技能参数
  - 关键词检测命中后：`get_skill_by_name()` → `skill.run()` → LLM 润色（tools=None）→ 返回
  - 若技能执行失败，自动回退到正常 LLM 流程
  - 修改文件：`chat_service.py`（+`_parse_skill_args` + 强制执行逻辑）、`llm_client.py`（`debug_log` 改为 `logger.info`）、`llm_stream.py`（payload/response debug 日志）
